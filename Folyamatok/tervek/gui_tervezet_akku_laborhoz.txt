GUI TERVEZET
AKKU LABOR / GOLDEN IMAGE ELŐKÉSZÍTŐ SZOFTVERHEZ

Verzió: 1.0
Cél: Olyan asztali GUI tervezet készítése, amely a jelenlegi laborfolyamatokhoz,
műszervezérléshez és akkutesztekhez illeszkedik.

Alaprendszer:
- Tápegység: Keithley 2220-30-1
- Elektronikus terhelés: Keithley 2380-120-60
- DMM: Keysight 34465A
- Opcionális relék / kapcsolóegység
- BQ kommunikáció nincs, a GUI a fizikai laborfolyamatokat vezérli

Fő célok:
1. A kezelő egyértelműen lássa az aktuális állapotot.
2. A kritikus paraméterek mindig legyenek szem előtt.
3. A mérési adatok folyamatosan mentésre kerüljenek.
4. Hiba, áramszünet, programösszeomlás vagy megszakítás esetén az addig elért eredmények
   maradjanak meg.
5. A futó folyamat grafikonokon és állapotjelzőkön követhető legyen.
6. A GUI alkalmas legyen:
   - teljes töltési ciklusra
   - kisütésre
   - relaxációra
   - kapacitástesztre
   - OCV-SOC mérésre
   - teljes golden image előkészítő ciklusok futtatására


1. JAVASOLT TECHNOLÓGIA
=======================

1.1 GUI keretrendszer
---------------------
Python alapú megvalósításhoz javasolt:

- PySide6 vagy PyQt6
- pyqtgraph a gyors élő grafikonokhoz
- matplotlib opcionálisan riport exporthoz
- sqlite3 helyi adatbázisnak
- csv / json folyamatos mentéshez
- threading vagy QThread a háttérfolyamatokhoz
- queue a GUI és a mérési motor közti üzenetváltáshoz

Javaslat:
- GUI: PySide6
- élő grafikon: pyqtgraph
- tartós naplózás: sqlite + csv párhuzamosan

Indok:

13.6 Flush stratégia
--------------------
Javaslat:

13.8 Megszakítás utáni helyreállítás
------------------------------------
Program induláskor a GUI ellenőrizze:
- van-e unfinished session
- van-e checkpoint
- a session status running / paused / fault / aborted maradt-e

Induláskor jelenjen meg egy recovery ablak:

„Találtunk egy be nem fejezett sessiont.
Session ID: ...
Állapot: ...
Utolsó mintavétel: ...
Szeretné:
1. csak megnyitni
2. riportként lezárni
3. folytatni
4. archiválni?”

Megjegyzés:
Nem minden folyamat folytatható automatikusan biztonságosan.
Ezért kétféle recovery kell:

A. Data recovery
- az addigi eredmények megmaradnak
- riport elkészíthető
- új teszt indítható

B. Process recovery
- csak akkor engedélyezett, ha a műszerek állapota újra biztonságosan validálható
- új precheck kell
- a felhasználó megerősítése kell

13.9 Programösszeomlás elleni minimum
-------------------------------------
Minimum elvárás:
- összeomlás esetén a legutolsó néhány minta veszhet el maximum
- az egész session nem veszhet el
- a session meta és eseménylog mindig megmaradjon
- a GUI újranyitásakor látható legyen, hogy mi történt

13.10 Vészleállítás után mentés

1. LOAD OFF
2. PSU OFF

4. active fault log

20. RÖVID ÖSSZEFOGLALÓ
======================

A javasolt GUI fő jellemzői:

1. Dashboard központú felépítés
   - nagy, jól látható élő paraméterek
   - gyors állapotáttekintés

2. Külön Műszerek nézet
   - kapcsolat
   - diagnosztika
   - kézi alapteszt

3. Külön Akkuprofil és Tesztterv nézet
   - paraméterezhetőség
   - reprodukálhatóság

4. Futó Teszt nézet
   - állapotgép
   - élő paraméterek
   - kezelői gombok

5. Grafikonok nézet
   - U, I, P, T, Ah, Wh, U_drop
   - export funkciók

6. Folyamatos, megszakításbiztos mentés
   - sqlite + csv
   - session metaadat
   - eseménynapló
   - checkpoint
   - recovery

7. Hiba esetén sem veszhet el az addigi mérés
   - session azonnali létrehozás
   - mintánkénti mentés
   - kritikus események azonnali flush
   - recovery ablak újraindításkor

Ez a GUI-tervezet jó alap lehet ahhoz, hogy a szoftvert lépésenként felépítsd,
először stabil mérési/logolási maggal, majd egyre kényelmesebb kezelőfelülettel.
