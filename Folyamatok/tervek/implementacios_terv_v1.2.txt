VEZETŐI ÖSSZEFOGLALÓ
================================================================================

A projekt célja egy Python alapú asztali laboralkalmazás, amely Keithley 2220-30-1
tápegységet, Keithley 2380-120-60 elektronikus terhelést és 2 db Keysight 34465A
DMM-et vezérel (DMM1: feszültség, DMM2: felületi hőmérséklet). A szoftver FIAMM
AGM/VRLA akkumulátorok laborfolyamatait automatizálja (töltés CC/CV soros Schottky
diódakompenzációval, CC kisütés, relaxáció, kapacitásteszt, OCV-SOC lépcsőzés).

Galvanikus leválasztás NINCS — PSU output OFF szintű.
Relé hardware NINCS — relay_matrix.py NEM KÉSZÜL, stub sem.
BQ34Z110/BQ34Z100 golden image előkészítő fizikai ciklusok BQ-kommunikáció nélkül.
PySide6 GUI, SQLite+CSV logolás, safety-first tervezés.

Jelenlegi állapot: kód nincs — Prog/ mappa üres, csak .gitkeep fájlok.
Minden specifikáció és a jelen v1.2 terv implementációra kész.

GUI stop modell:
  - Minden műszerhozzáférés kizárólag measurement worker thread-ben történik.
  - GUI közvetlenül SOHA nem hív driver metódust.
  - stop_requested flag: kérés a mérési motor normál leállítására

4.2 Állapotgép (töltés) [R1]
------------------------------
  INIT
  └─> PRECHECK                     ← akkufeszültség tartomány ellenőrzés [R5]
      └─> PSU_PRESET               ← PSU mód ellenőrzés, árambeállítás
          └─> CHARGE_RAMP          ← CONNECT_RELAY eltávolítva [R1]
              └─> CHARGE_CC        ← áramlimitben dolgozik
                  └─> CHARGE_CV_DMM_CONTROL  ← DMM-kompenzált CV
                      └─> TAPER_HOLD  ← taper feltétel tartása [R9]
                          └─> CHARGE_DONE
                              └─> RELAX / DONE
  Bármely állapotból: → FAULT / SAFE_OFF

  CHARGE_CV_DMM_CONTROL (minden control_update_period_s-ban):
    error = U_batt_target(T) - U_batt_DMM
    ha |error| <= deadband_V: nincs változtatás
    ha error > deadband:  step = min(error, max_step_up_V);  U_psu_set += step
    ha error < -deadband: step = min(|error|, max_step_down_V); U_psu_set -= step
    U_psu_set = max(U_psu_set, 0)
    U_psu_set = min(U_psu_set, psu_limits_by_mode[mode].max_voltage_V)
    U_psu_set = min(U_psu_set, U_batt_target + max_expected_series_drop)
    Hőkompenzált cél (ha ENABLED): lásd 4.10

4.3 Állapotgép (kisütés)
--------------------------
  INIT
  └─> PRECHECK (PSU OFF ellenőrzés)
      └─> DISCHARGE_CC_SETUP
          └─> DISCHARGE_CC_RUN
              └─> DISCHARGE_DONE
                  └─> RELAX_AFTER_DISCHARGE

  Stop feltételek:
    U_batt_DMM <= terminate_voltage_pack_V
    elapsed_time >= max_discharge_time_h
    safety hiba

4.4 Állapotgép (BQ learning fizikai ciklus)
--------------------------------------------
  INIT → PRECHECK
  Ciklus 1:
    FULL_CHARGE_1 → RELAX_AFTER_CHARGE_1 (min 2h) →
    DISCHARGE_C5_1 → RELAX_AFTER_DISCHARGE_1 (min 5h)
  Ciklus 2:
    FULL_CHARGE_2 → RELAX_AFTER_CHARGE_2 (min 2h) →
    DISCHARGE_C5_2 → RELAX_AFTER_DISCHARGE_2 (min 5h)
  → MANUAL_BQ_CHECKPOINT
  → DONE → riport

4.5 Állapotgép (OCV-SOC lépcsős teszt)
----------------------------------------
  INIT → PRECHECK → Teljes töltés + 5h relax
  capacity_fraction_index = 100%   ← "SOC_estimate" helyett [R24 ref]
  OCV_mérés log
  Ciklus amíg capacity_fraction_index > 0:
    1. Load CC = C/10 (vagy C/20)
    2. Ah integrálás → ha removed_Ah >= step_Ah → Load OFF
    3. Relax (1–2h, kulcspontokon 5h)
    4. DMM OCV mérés (NPLC 100)
    5. Log: capacity_fraction_index, removed_Ah, OCV, dVdt, T
       Megjegyzés: SOC_estimate_method = "coulomb_counting_from_measured_capacity"
    6. capacity_fraction_index -= nominal_removed_capacity_step_percent (alap: 5%)
    Ha feszültség eléri terminate_voltage: ACTUAL_EMPTY_REACHED

4.7 BatteryProfile dataclass [R2]
-----------------------------------
@dataclass
class BatteryProfile:
    battery_name: str
    manufacturer: str
    model: str
    chemistry: str = "AGM_VRLA"
    profile_source: str = ""
    nominal_voltage_V: float = 12.0
    cell_count: int = 6
    nominal_capacity_Ah: float = 0.0        # kötelező — validáció: > 0
    charge_voltage_per_cell_V: float = 2.40
    terminate_voltage_per_cell_V: float = 1.80
    float_voltage_per_cell_V: float = 2.27
    max_charge_current_A: float | None = None
    taper_current_A: float | None = None
    temp_comp_mV_per_cell_per_degC: float = -2.5
    test_temperature_min_C: float = 15.0
    test_temperature_max_C: float = 30.0

    # Számított @property-k — FIGYELEM: C10_discharge_current_A ≠ C-ráta jelölés [R2]
    # charge_voltage_pack_V      = charge_voltage_per_cell_V * cell_count
    # terminate_voltage_pack_V   = terminate_voltage_per_cell_V * cell_count
    #
    # C10_discharge_current_A    = nominal_capacity_Ah / 10   ← kisütőáram (10h ráta)
    # C5_discharge_current_A     = nominal_capacity_Ah / 5
    # C20_discharge_current_A    = nominal_capacity_Ah / 20
    #
    # effective_max_charge_A:
    #   ha max_charge_current_A megadva: azt használja
    #   különben: 0.25 * nominal_capacity_Ah  ← FIAMM 0.25C = 0.25 × Ah [R2]
    #
    # effective_taper_A:
    #   ha taper_current_A megadva: azt használja
    #   különben: 0.03 * nominal_capacity_Ah  ← FIAMM 0.03C = 0.03 × Ah [R2]

  MEGJEGYZÉS a C-ráta jelölésről [R2]:
    A FIAMM dokumentumban "0,25 × C10" azt jelenti: 25%-a a névleges kapacitásnak [Ah].

4.9 Safety manager [R1, R15]
------------------------------
Minden mérési ciklusban ellenőriz:
  battery_voltage_V > batt_absolute_max_V            → BATTERY_OVERVOLTAGE
  battery_voltage_V < batt_absolute_min_V            → BATTERY_UNDERVOLTAGE
  battery_temperature_C > max_temperature_C          → BATTERY_TEMPERATURE_HIGH
  abs(charge_current_A) > max_charge_current * (1+tol) → CHARGE_CURRENT_TOO_HIGH
  abs(discharge_current_A) > max_discharge_current * (1+tol) → DISCHARGE_CURRENT_TOO_HIGH
  load_power_W > max_load_power_W                    → LOAD_POWER_LIMIT
  U_drop > max_allowed_series_drop                   → SERIES_DROP_TOO_HIGH
  U_drop < -0.05                                     → NEGATIVE_SERIES_DROP
  not dmm_voltage_valid                              → DMM_FEEDBACK_LOST
  psu_output_on AND load_input_on (nem RELAX)        → CONCURRENT_PSU_LOAD_ON
  charged_Ah > nominal_Ah * max_charge_Ah_factor     → MAX_CHARGE_AH_REACHED
  elapsed_charge_time > max_charge_time              → MAX_CHARGE_TIME_REACHED

Hibaosztályozás [R15]:
  FAULT_FATAL — azonnali emergency_stop:
    DMM_FEEDBACK_LOST (töltés/kisütés közben)
    BATTERY_OVERVOLTAGE
    CONCURRENT_PSU_LOAD_ON
    LOAD_POWER_LIMIT
    PSU_COMM_LOST (töltés közben)
    MAX_CHARGE_TIME_REACHED
    MAX_CHARGE_AH_REACHED

  WARNING — logolva, GUI figyelmeztetés, mérés folytatódik:
    TEMPERATURE_OUT_OF_RANGE_WARNING
    HEADROOM_APPROACHING
    REDUCED_CHARGE_CURRENT_DUE_TO_PSU_LIMIT
    PSU_LEAKAGE_SUSPECTED
    INTEGRATION_FALLBACK_TO_SETPOINT

  INFO — logolva, GUI info:
    TAPER_ENTERED
    RELAX_STARTED
    CHECKPOINT_WRITTEN
    BQ_CHECKPOINT_REACHED

Vészleállítás sorrendja (safety_first) [R1]:
  1. load.input_off()       → LOAD OFF  (try/except: hiba logolva, folytatódik)
  2. psu.all_outputs_off()  → PSU OFF   (try/except: hiba logolva, folytatódik)
  NINCS relay.safe_open() — relé hardware nem létezik. [R1]
  3. isolation_state = PSU_OUTPUT_OFF_ONLY
  4. log EMERGENCY_STOP + reason
  5. checkpoint mentés (kritikus esemény → azonnali FLUSH)
  6. GUI fault screen

  MEGJEGYZÉS: safe_off minden lépése külön try/except blokkban fut.
  Ha az első lépés hibát dob, a második akkor is végrehajtódik.

4.11 Taper feltétel — formális definíció [R9]
----------------------------------------------
TAPER_HOLD állapotba lépés feltétele (MINDEN feltétel egyszerre):
  U_batt_DMM >= (U_batt_target - cv_voltage_tolerance_V)
  AND I_charge <= taper_current_A
  AND state == CHARGE_CV_DMM_CONTROL
  AND dmm_voltage_valid == True

  cv_voltage_tolerance_V = cell_count * cv_voltage_tolerance_V_per_cell
  (12V pack: 6 * 0.003 = 0.018 V; 24V pack: 12 * 0.003 = 0.036 V)

TAPER_HOLD timer nullázódik (reset), ha bármely feltétel megsérül:
  I_charge > taper_current_A * taper_current_tolerance_factor
  OR U_batt_DMM < (U_batt_target - cv_voltage_tolerance_V)
  OR dmm_voltage_valid == False

CHARGE_DONE, ha:
  taper_hold_timer_s >= taper_hold_s  (konfig: 600 s)

Log mezők:
  taper_timer_s           — aktuális taper timer érték
  taper_cond_voltage_ok   — bool
  taper_cond_current_ok   — bool

4.12 Precheck akkufeszültség-tartomány [R5]
--------------------------------------------

12V / 6 cella pack:
  U_batt < 6.0 V:               ERROR — mélykisült/rossz akku, töltés TILTOTT
  6.0 V ≤ U_batt < 10.5 V:      WARNING — mélykisült, recovery mód szükséges
                                   normál CC/CV profil nem indítható

  21.0 V ≤ U_batt ≤ 29.0 V:      OK

4.17 FIAMM-specifikus konfiguráció — sablonszámítás [R2, R4]
--------------------------------------------------------------
MEGJEGYZÉS a C-ráta számításhoz:
  A FIAMM "0,25 × C10" és "0,03 × C10" jelölésben C10 = névleges kapacitás [Ah],
  nem a C/10 kisütési áram. Az implementációban:
    effective_max_charge_A = 0.25 * nominal_capacity_Ah  [A]
    effective_taper_A      = 0.03 * nominal_capacity_Ah  [A]

12V pack (cell_count=6), X = nominal_capacity_Ah:
  charge_voltage_pack_V    = 6 × 2.40 = 14.40 V
  terminate_voltage_pack_V = 6 × 1.80 = 10.80 V
  float_voltage_pack_V     = 6 × 2.27 = 13.62 V
  max_charge_current_A     = 0.25 × X  [A]
  taper_current_A          = 0.03 × X  [A]
  C5_discharge_A           = X / 5   [A]
  C10_discharge_A          = X / 10  [A]
  C20_discharge_A          = X / 20  [A]
  U_psu_set_max (INDEPENDENT/PARALLEL) = min(14.40 + 0.90, 30.0) = 15.30 V

24V pack (cell_count=12), SERIES mód: [R4]
  charge_voltage_pack_V    = 12 × 2.40 = 28.80 V
  terminate_voltage_pack_V = 12 × 1.80 = 21.60 V
  float_voltage_pack_V     = 12 × 2.27 = 27.24 V
  max_charge_current_A     = 0.25 × X  [A]
  taper_current_A          = 0.03 × X  [A]
  U_psu_set_max (SERIES)   = min(28.80 + 1.00, 60.0) = 29.80 V
  MEGJEGYZÉS SERIES módban [R4]:
    - Rendelkezésre álló PSU max feszültség: 60 V
    - A 29.80 V cél bőséges headroommal elérhető (60V limitből)
    - 24V packnál a fő korlát NEM a feszültség, hanem az 1.5A áramlimit
    - 6 Ah feletti 24V pack esetén a 0.25C ráta nem teljesíthető 1.5A mellett

Példa 12V / 7 Ah:
  max_charge_current_A = 0.25 × 7.0 = 1.75 A
  taper_current_A      = 0.03 × 7.0 = 0.21 A
  C5 = 1.40 A, C10 = 0.70 A, C20 = 0.35 A

Példa 12V / 18 Ah:
  max_charge_current_A = 0.25 × 18.0 = 4.50 A
  taper_current_A      = 0.03 × 18.0 = 0.54 A
  C5 = 3.60 A, C10 = 1.80 A, C20 = 0.90 A

4.18 PSU kombinációs üzemmód — mód-függő határok [R4]
-------------------------------------------------------
A 2220-30-1 három üzemmódban használható. Mód-függő PSU limitek:

  INDEPENDENT: INST:COMB:OFF
    max_voltage_V: 30.0
    max_current_A: 1.5
    Elsődleges használat: 12V pack ≤ 6 Ah

  PARALLEL: INST:COMB:PAR
    max_voltage_V: 30.0
    max_current_A: 3.0
    Elsődleges használat: 12V pack 7–12 Ah

  SERIES: INST:COMB:SER
    max_voltage_V: 60.0
    max_current_A: 1.5
    Elsődleges használat: 24V pack (minden kapacitás)

Headroom számítás (mód-függő) [R4]:
  U_psu_required = U_batt_target + max_expected_series_drop
  if U_psu_required > psu_limits_by_mode[mode].max_voltage_V:
      WARNING: HEADROOM_APPROACHING
  if tartósan (60 s) fennáll:
      FAULT_FATAL: PSU_HEADROOM_LIMIT

Kompatibilitási táblázat:

  Pack   | Kapacitás | 0.25C áram  | PSU mód    | Megjegyzés
  -------+-----------+-------------+------------+------------------------------------------
  12V    | ≤ 6 Ah    | ≤ 1.5 A     | INDEPENDENT | CH1 egyedül elegendő
  12V    | 7–12 Ah   | 1.75–3.0 A  | PARALLEL    | 3A max → 12Ah-ig 0.25C teljesíthető
  12V    | > 12 Ah   | > 3 A       | PARALLEL+cs | 3A limit → csökkentett C-ráta
  24V    | bármely   | változó     | SERIES      | Feszültség miatt kötelező; max 1.5 A
  24V    | > 6 Ah    | > 1.5 A     | SERIES+cs   | 1.5A korlát → alacsonyabb C-ráta

Konkrét példák:
  12V /  7 Ah: INDEPENDENT — 1,75 A > 1,5 A FIGYELEM! [R3]
               → PARALLEL ajánlott (1,75 A biztonságosan)
               → VAGY INDEPENDENT max 1,40 A (0.20C)
  12V / 12 Ah: PARALLEL — 0.25×12 = 3.0 A, pontosan a maximum
  12V / 18 Ah: PARALLEL + aránycsökkentés — 3A / 18Ah = 0.167C max
  24V /  6 Ah: SERIES — 0.25×6 = 1.5 A, pontosan a maximum
  24V / 18 Ah: SERIES + aránycsökkentés — 1.5A / 18Ah = 0.083C max

PSU mód váltás biztonsági sorrend:
  1. GUI dialógus megerősítés
  2. PSU all_outputs_off()
  3. INST:COMB:PAR / INST:COMB:SER parancs
  4. INST:COMB? visszaolvasás — ellenőrzés
  5. Precheck: MEAS:VOLT? plausibility check
     SERIES módban: ha readback < 5V (akku csatlakoztatva esetén gyanús)
     PARALLEL módban: ha readback > 30V → gyanús (soros bekötés?)
  6. session_meta.json: wiring_validation_timestamp, measured_precheck_voltage, validated_mode
  7. Konfiguráció mentése

4.19 GUI réteg
---------------

Kommunikáció: QThread + Queue + Qt signals

5.1 DMM-feedback elvesztése
-----------------------------
KOCKÁZAT: DMM LAN-kapcsolat megszakad → szabályozóhurok vak.
KÖVETKEZMÉNY: PSU folytatja a töltést ismeretlen akkufeszültségen → túltöltés.
MITIGÁCIÓ:
  - dmm_voltage_valid = False → FAULT_FATAL: DMM_FEEDBACK_LOST → emergency_stop
  - Minden DMM lekérdezésnél try/except + timeout (10000 ms, NPLC100-nál 30000 ms)
  - Irreális ugrás (>2V egyetlen mintában) → invalid reading → szintén FAULT_FATAL

5.2 Akkufeszültség túllépés
-----------------------------
KOCKÁZAT: Rosszul számított series drop, hibás konfiguráció.
KÖVETKEZMÉNY: AGM/VRLA gázolás, szivárgás.
MITIGÁCIÓ:
  - batt_absolute_max_V folyamatosan ellenőrizve
  - Ha U_batt_DMM > batt_absolute_max_V: FAULT_FATAL: BATTERY_OVERVOLTAGE

5.3 Tápegység headroom-limit (Schottky dióda esetén) [R4]
----------------------------------------------------------
KOCKÁZAT: Schottky esés 0,4–0,8 V áramfüggő.

12V pack (INDEPENDENT/PARALLEL, max 30 V):
  U_psu_required = 14.40 + 0.90 = 15.30 V → bőséges headroom (30 V-ig)
  U_psu_set_max = 15.30 V (kemény limit)

24V pack (SERIES, max 60 V): [R4 javítás]
  U_psu_required = 28.80 + 1.00 = 29.80 V → bőséges headroom (60 V-ig)
  U_psu_set_max = 29.80 V
  MEGJEGYZÉS: 24V packnál SERIES módban a feszültség headroom NEM kritikus.
  A fő korlát az 1.5A áramlimit, nem a PSU max feszültség.
  24V pack INDEPENDENT/PARALLEL módban NEM engedélyezett.

MITIGÁCIÓ:
  - U_psu_set_max: mód-függő korlát (lásd 4.18)
  - HEADROOM_APPROACHING WARNING: ha 60s-en belül nem éri el a célt
  - FAULT_FATAL: PSU_HEADROOM_LIMIT (tartós, >60s)

5.4 Soros dióda / feszültségesés
----------------------------------
KOCKÁZAT: U_drop áram- és hőmérsékletfüggő.
KÖVETKEZMÉNY: PSU set értéke dinamikusan változik.
MITIGÁCIÓ:
  - U_drop = U_psu_readback - U_batt_DMM folyamatosan logolva
  - Ha U_drop > max_allowed_series_drop: FAULT_FATAL: SERIES_DROP_TOO_HIGH
  - Ha U_drop < -0.05 V: WARNING: NEGATIVE_SERIES_DROP (bekötési hiba gyanú)
  - A CV szabályozás DMM-alapú, nem PSU-readback alapú (emiatt helyes)

5.5 Egyszerre aktív PSU és elektronikus terhelés
--------------------------------------------------
KOCKÁZAT: Versenyfeltétel → PSU ON + Load ON egyszerre.
MITIGÁCIÓ:
  - Safety manager: NOT (psu_output_on AND load_input_on) → FAULT_FATAL
  - Állapotgép garantálja: töltés előtt load.input_off(); kisütés előtt psu.all_outputs_off()

5.6 Nincs relé — galvanikus leválasztás hiánya [R1]
----------------------------------------------------
KOCKÁZAT: PSU output OFF nem teljes galvanikus leválasztás.
  Schottky dióda fordított szivárgása típus- és hőmérsékletfüggő.
KÖVETKEZMÉNY: OCV mérések minimálisan torzulhatnak.
MITIGÁCIÓ:
  - isolation_state = PSU_OUTPUT_OFF_ONLY (minden logban és riportban jelölve)
  - PSU OFF után DMM current readback: ha I_psu > 5 mA → WARNING: PSU_LEAKAGE_SUSPECTED
  - Riportban kötelező: galvanic_isolation: false
  - OCV mérésnél a szivárgás hatása FELTÉTELEZÉS: hosszú relaxnál (5h) elhanyagolható,
    de konkrét Schottky típus nélkül NEM IGAZOLHATÓ általánosan.
  Nincs relay_matrix stub — ha a jövőben relé kerül a rendszerbe, az külön
  feature branch és terv.

5.7 Mérési egységhibák
------------------------

MITIGÁCIÓ:
  - Minden driver SI egységben ad vissza (V, A, W)

  - Egységteszt: C/5 számítás 7 Ah → 1.40 A

5.8 Timeout és kommunikációs hiba
