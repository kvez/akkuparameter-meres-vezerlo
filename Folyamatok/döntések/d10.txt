JAVASOLT TESZTFÓKUSZ
====================

1. event_ready signal
---------------------

Teszt:

  - mock runner on_event callbacket hív
  - worker event_ready signal elsül
  - payload tartalmaz event_code-ot

Elvárt:

  event_code == "EMERGENCY_STOP"
  vagy más tesztesemény


2. checkpoint_reached signal
----------------------------

Teszt:

  - event_code == "MANUAL_BQ_CHECKPOINT_REACHED"
  - worker event_ready is elsül
  - worker checkpoint_reached is elsül

Ez fontos, mert ez volt a konkrét nyitott pont.


3. status_changed
-----------------

Teszt:

  - worker RUNNING státuszt küld
  - worker CHECKPOINT_STOPPED státuszt küld
  - worker FAULT státuszt küld

Nem kell a QLabel textet ellenőrizni.
Elég a signal payload.


4. stop / emergency slotok
--------------------------

Teszt:

  worker.request_stop()
    → runner.request_stop_called == True

  worker.request_emergency_stop("USER_EMERGENCY_STOP")
    → runner.request_emergency_stop_called == True
    → reason átmegy


5. finished / fault
-------------------

Teszt:

  runner.run() TestResult("DONE")-t ad
    → worker.finished signal elsül

  runner.run() exceptiont dob
    → worker.fault signal elsül
    → opcionálisan emergency_stop request megtörténik


MIT NEM TESZTELNÉK MÉG UNITBAN?
===============================

MVP-ben nem tesztelném külön:

  - LivePanel konkrét layoutját
  - eseménylista vizuális sorait
  - checkpoint panel checkboxait
  - status bar pontos színét
  - grafikon widget rajzolását
  - tabváltás vizuális működését

Ezeket manuális smoke teszttel ellenőrizném:

  python Prog/main.py


KÉSŐBBI BŐVÍTÉS
===============

Ha a GUI stabilabb lesz, akkor lehet hozzáadni pár könnyű widget tesztet:

  - LivePanel.append_event(event) növeli az eseménysorok számát
  - CheckpointPanel.show_checkpoint(event) beállítja a szöveget és engedélyezi a gombot
  - MainWindow._on_checkpoint_reached(event) átvált checkpoint panelre

De ezek legyenek később, ne most blokkolják az implementációt.


JAVASOLT DÖNTÉS A TERVBE
========================

FÁZIS 6B tesztstratégia:

  Elsődleges automatizált lefedés:
    worker-szintű signal/slot tesztek

  Widget/panel tesztek:
    nem kötelezőek MVP-ben

  Manuális GUI smoke teszt:
    kötelező

Smoke teszt checklist:

  - python Prog/main.py elindul
  - Start gomb után RUNNING állapot látszik
  - sample_ready frissíti az élő értékeket
  - event_ready esemény bekerül az eseménylistába
  - MANUAL_BQ_CHECKPOINT_REACHED esetén checkpoint panel aktiválódik
  - Stop gomb működik
  - Emergency Stop gomb működik
  - FAULT esetén fault label/status frissül


VÉGSŐ VÁLASZ
============

B) Csak worker-szintű tesztek

Kiegészítve:

  - legyen Prog/tests/gui/ worker tesztcsomag
  - checkpoint_reached signal külön legyen tesztelve
  - event_ready signal külön legyen tesztelve
  - widgetekre most elég manuális smoke teszt
  - később lehet célzott panel teszteket hozzáadni, ha stabilizálódik a GUI
