Javaslat: 1 + 2 kombinációja.

FÁZIS 5-ben a TestRunner legyen szimpla Python osztály, GUI-független üzleti logikával,
de a run() metódusa lehet blokkoló, stop flag-ekkel.

Tehát:

  - Ne legyen benne GUI.
  - Ne legyen benne QThread.
  - Ne legyen benne asyncio.
  - Ne legyen benne saját main() belépési pont.
  - Ne hozzon létre saját threadet.
  - Kívülről lehessen hívni CLI-ből, tesztből vagy később GUI worker threadből.
  - A run() blokkolhat, mert FÁZIS 5-ben ez a legegyszerűbb és legjobban tesztelhető.
  - A stop_requested és emergency_stop_requested flag-ek legyenek benne.

MŰKÖDÉSI MODELL
===============

FÁZIS 5-ben CLI / teszt / mock környezetből:

  runner = TestRunner(...)
  result = runner.run(test_plan)

Ez blokkol, amíg:

  - a teszt be nem fejeződik
  - stop_requested nem érkezik
  - emergency_stop_requested nem érkezik
  - safety fault nem történik


FÁZIS 6-ban GUI-ból:

  A GUI nem közvetlenül hívogatja a drivereket.
  A GUI létrehoz egy worker threadet.
  A worker threadben fut:

    runner.run(test_plan)

  A GUI gombjai csak ezt hívják:

    runner.request_stop()
    runner.request_emergency_stop("USER_EMERGENCY_STOP")

Így a TestRunner ugyanaz marad CLI-ben, mock tesztben és GUI alatt is.


MIÉRT EZ A LEGJOBB?
===================

1. Egyszerűen tesztelhető
-------------------------

Mock driverekkel simán futtatható:

  result = runner.run(mock_test_plan)

Nem kell Qt event loop.
Nem kell asyncio.
Nem kell thread-szinkronizáció az unit tesztekben.


2. GUI-független
----------------

A TestRunner nem tud PySide6-ról.
Ez jó, mert az üzleti logika nem keveredik UI logikával.


3. FÁZIS 6-ban könnyen threadbe tehető
--------------------------------------

A blokkoló run() nem baj.
Sőt, GUI-nál pont ez a jó:

  QThread / Worker:
      runner.run(test_plan)

A GUI pedig signal/queue alapon kap állapotfrissítést.


4. A stop logika explicit
-------------------------

A runner ciklus minden állapotlépés előtt ellenőrzi:

  if self.emergency_stop_requested:
      self.emergency_stop(self.emergency_stop_reason)
      return TestResult(status="FAULT")

  if self.stop_requested and self._safe_to_stop():
      self.graceful_stop()
      return TestResult(status="STOPPED")

Ez egyszerű és jól átlátható.


5. asyncio most felesleges
--------------------------

Az asyncio FÁZIS 5-ben túl sok komplexitást hozna:

  - VISA / PyVISA hívások jellemzően blokkolók
  - műszerkommunikáció nem natív async
  - PySide6 integráció külön odafigyelést igényelne
  - tesztek bonyolultabbak lennének

Ezért asyncio-t most nem javaslok.


JAVASOLT DÖNTÉS A TERVBE
========================

FÁZIS 5 TestRunner futtatási modell:

  A TestRunner tiszta Python üzleti-logikai osztály.
  Nem tartalmaz GUI-kódot, QThread-et, asyncio-t vagy main() belépési pontot.
  A run(test_plan) metódus blokkoló módon fut, és kívülről hívható CLI-ből,
  unit tesztből vagy később GUI worker threadből.
  A leállítás request_stop() és request_emergency_stop(reason) metódusokon keresztül
  történik. Ezek flag-eket állítanak, amelyeket a run() ciklus minden állapotlépés
  előtt és után ellenőriz.
  A TestRunner nem hoz létre saját threadet. A threading a hívó réteg felelőssége.


JAVASOLT OSZTÁLYVÁZ
===================

class TestRunner:
    def __init__(
        self,
        instrument_manager,
        safety_manager,
        logger,
        profile,
        config,
        charge_controller,
        discharge_controller,
        relax_controller,
    ):
        self.instrument_manager = instrument_manager
        self.safety = safety_manager
        self.logger = logger
        self.profile = profile
        self.config = config
        self.charge_controller = charge_controller
        self.discharge_controller = discharge_controller
        self.relax_controller = relax_controller

        self.stop_requested = False
        self.emergency_stop_requested = False
        self.emergency_stop_reason = ""

        self.current_step = None
        self.status = "IDLE"

    def request_stop(self) -> None:
        self.stop_requested = True

    def request_emergency_stop(self, reason: str = "USER_EMERGENCY_STOP") -> None:
        self.emergency_stop_requested = True
        self.emergency_stop_reason = reason

    def run(self, test_plan):
        self.status = "RUNNING"

        try:
            for step in test_plan.steps:
                self.current_step = step

                if self.emergency_stop_requested:
                    return self._emergency_stop_and_result(self.emergency_stop_reason)

                if self.stop_requested:
                    return self._graceful_stop_and_result("USER_STOP_REQUESTED")

                result = self._run_step_blocking(step)

                if result.status == "FAULT":
                    return result

            self.status = "DONE"
            return TestResult(status="DONE")

        except Exception as exc:
            return self._emergency_stop_and_result(f"UNHANDLED_EXCEPTION: {exc}")

    def _run_step_blocking(self, step):
        if step.kind == "CHARGE":
            controller = self.charge_controller
        elif step.kind == "DISCHARGE":
            controller = self.discharge_controller
        elif step.kind == "RELAX":
            controller = self.relax_controller
        else:
            raise ValueError(f"Unknown test step kind: {step.kind}")

        while not controller.is_finished:
            if self.emergency_stop_requested:
                return self._emergency_stop_and_result(self.emergency_stop_reason)

            if self.stop_requested and controller.safe_to_stop:
                return self._graceful_stop_and_result("USER_STOP_REQUESTED")

            controller.advance(dt_s=self.config.runner_tick_s)

            self.logger.write_sample(controller.current_sample)
            self.logger.maybe_flush()
            self.logger.maybe_checkpoint()

        return TestResult(status="STEP_DONE")

    def _emergency_stop_and_result(self, reason: str):
        self.status = "FAULT"
        self.instrument_manager.safe_all_off()
        self.logger.log_event("EMERGENCY_STOP", reason)
        self.logger.write_checkpoint(status="FAULT", reason=reason)
        self.logger.flush_all()
        return TestResult(status="FAULT", reason=reason)

    def _graceful_stop_and_result(self, reason: str):
        self.status = "STOPPED"
        self.instrument_manager.safe_all_off()
        self.logger.log_event("STOPPED", reason)
        self.logger.write_checkpoint(status="STOPPED", reason=reason)
        self.logger.flush_all()
        return TestResult(status="STOPPED", reason=reason)


FONTOS IMPLEMENTÁCIÓS SZABÁLYOK
================================

1. TestRunner ne importáljon PySide6-ot.

2. TestRunner ne hozzon létre threadet.

3. TestRunner ne ismerje a GUI-t.

4. TestRunner csak kontrollereket, logger-t, safety-t és instrument_manager-t hívjon.

5. Minden hosszabb ciklusban legyen:

   - emergency_stop_requested ellenőrzés
   - stop_requested ellenőrzés
   - checkpoint lehetőség
   - logger flush lehetőség

6. Minden kivétel safety irányba essen:

   try/except Exception:
       emergency_stop
       fault result

7. A CLI wrapper külön fájl lehet később:

   run_test_cli.py

   De ez ne legyen a TestRunner belsejében.


FÁZIS 5 TESZT FUTTATÁS GUI NÉLKÜL
=================================

CLI / mock példa:

  profile = BatteryProfile(...)
  instrument_manager = MockInstrumentManager(...)
  safety = SafetyManager(...)
  logger = SessionLogger(...)
  runner = TestRunner(...)

  result = runner.run(test_plan)

  assert result.status in ("DONE", "FAULT", "STOPPED")


Későbbi GUI példa:

  worker thread:
      result = runner.run(test_plan)

  GUI stop gomb:
      runner.request_stop()

  GUI emergency stop gomb:
      runner.request_emergency_stop("USER_EMERGENCY_STOP")


KONKLÚZIÓ
=========

Választás:

  1. Szimpla Python osztály
  +
  2. Blocking futás + stop flag

Ez a legjobb FÁZIS 5-höz.

Ne legyen asyncio.
Ne legyen belső threading.
Ne legyen main() a TestRunnerben.

A TestRunner legyen tiszta, blokkoló, mockolható üzleti logika, amit kívülről
lehet futtatni CLI-ből, unit tesztből vagy később GUI worker threadből.
