Igen, ez jó irány, mehet.

A struktúra FÁZIS 5-höz tiszta és elég egyszerű:

  - TestType: magas szintű teszttípus
  - StepKind: végrehajtható lépés típusa
  - TestStep: egy konkrét lépés a tervben
  - TestPlan: lépéssorozat gyártómetódusokkal
  - TestResult: futás végeredménye
  - TestRunnerConfig: runner alapbeállítások

Ez jól illeszkedik ahhoz a döntéshez, hogy a TestRunner GUI-független, blokkoló,
kívülről hívható üzleti logika legyen.


JAVASOLT KISEBB PONTOSÍTÁSOK
============================

1. TestStep maradhat egyszerű
-----------------------------

Ez így jó:

  @dataclass
  class TestStep:
      kind: StepKind
      label: str

Mivel a kontrollerek kívülről jönnek előre inicializálva, nem kell controller példányt
a TestStep-be tenni.

Jó döntés, mert:

  - a TestPlan csak sorrendet ír le
  - a TestRunner dönti el, melyik controller fut egy adott StepKind alapján
  - könnyű tesztelni
  - nem keveredik a terv és az objektum-injektálás


2. TestPlan gyártómetódusok
---------------------------

A két staticmethod jó:

  TestPlan.characterization()
  TestPlan.bq_learning_physical()

Én ezekben konkrét label-eket használnék:

  characterization:
    CHARGE     "charge"
    RELAX      "relax_after_charge"
    DISCHARGE  "discharge"
    RELAX      "relax_after_discharge"

  bq_learning_physical:
    CHARGE     "charge_1"
    RELAX      "relax_after_charge_1"
    DISCHARGE  "discharge_1"
    RELAX      "relax_after_discharge_1"
    CHARGE     "charge_2"
    RELAX      "relax_after_charge_2"
    DISCHARGE  "discharge_2"
    RELAX      "relax_after_discharge_2"
    MANUAL_CHECKPOINT "manual_bq_checkpoint"

Ez jó lesz loghoz, riporthoz és checkpointhez is.


3. TestResult státusz
---------------------

A status: str működik, de jobb lenne enumot használni.

Opció A — most marad string:
  status: str = "DONE" / "FAULT" / "STOPPED"

Ez egyszerűbb és gyorsabb.

Opció B — tisztább:
  class TestStatus(Enum):
      DONE = "DONE"
      FAULT = "FAULT"
      STOPPED = "STOPPED"

  @dataclass
  class TestResult:
      status: TestStatus
      reason: str = ""
      total_charge_ah: float = 0.0
      total_discharge_ah: float = 0.0

FÁZIS 5-ben mindkettő jó.

Ha gyorsan akarsz haladni:
  maradjon string.

Ha hosszabb távra tisztábbat akarsz:
  legyen TestStatus enum.


4. TestRunnerConfig
-------------------

Ez jó:

  @dataclass
  class TestRunnerConfig:
      runner_tick_s: float = 2.0
      test_name: str = "unnamed"

Esetleg később bővíthető:

  checkpoint_every_s: float = 10.0
  sample_every_tick: bool = True

De most nem szükséges, mert ezek lehetnek logger configban is.


5. Enum értékek
---------------

Javaslom explicit string értékkel definiálni őket, ne auto()-val.

Példa:

  class TestType(Enum):
      CHARACTERIZATION = "CHARACTERIZATION"
      BQ_LEARNING_PHYSICAL = "BQ_LEARNING_PHYSICAL"

  class StepKind(Enum):
      CHARGE = "CHARGE"
      DISCHARGE = "DISCHARGE"
      RELAX = "RELAX"
      MANUAL_CHECKPOINT = "MANUAL_CHECKPOINT"

Indok:

  - logban olvashatóbb
  - JSON/CSV/session_meta egyszerűbb
  - checkpointben stabilabb
  - nem függ az enum sorrendtől

Én ezt választanám.


JAVASOLT VÉGLEGES 1. SZEKCIÓ
============================

from __future__ import annotations

from dataclasses import dataclass
from enum import Enum


class TestType(Enum):
    CHARACTERIZATION = "CHARACTERIZATION"
    BQ_LEARNING_PHYSICAL = "BQ_LEARNING_PHYSICAL"


class StepKind(Enum):
    CHARGE = "CHARGE"
    DISCHARGE = "DISCHARGE"
    RELAX = "RELAX"
    MANUAL_CHECKPOINT = "MANUAL_CHECKPOINT"


@dataclass(frozen=True)
class TestStep:
    kind: StepKind
    label: str


@dataclass(frozen=True)
class TestPlan:
    test_type: TestType
    steps: list[TestStep]

    @staticmethod
    def characterization() -> "TestPlan":
        return TestPlan(
            test_type=TestType.CHARACTERIZATION,
            steps=[
                TestStep(StepKind.CHARGE, "charge"),
                TestStep(StepKind.RELAX, "relax_after_charge"),
                TestStep(StepKind.DISCHARGE, "discharge"),
                TestStep(StepKind.RELAX, "relax_after_discharge"),
            ],
        )

    @staticmethod
    def bq_learning_physical() -> "TestPlan":
        return TestPlan(
            test_type=TestType.BQ_LEARNING_PHYSICAL,
            steps=[
                TestStep(StepKind.CHARGE, "charge_1"),
                TestStep(StepKind.RELAX, "relax_after_charge_1"),
                TestStep(StepKind.DISCHARGE, "discharge_1"),
                TestStep(StepKind.RELAX, "relax_after_discharge_1"),
                TestStep(StepKind.CHARGE, "charge_2"),
                TestStep(StepKind.RELAX, "relax_after_charge_2"),
                TestStep(StepKind.DISCHARGE, "discharge_2"),
                TestStep(StepKind.RELAX, "relax_after_discharge_2"),
                TestStep(StepKind.MANUAL_CHECKPOINT, "manual_bq_checkpoint"),
            ],
        )


@dataclass
class TestResult:
    status: str
    reason: str = ""
    total_charge_ah: float = 0.0
    total_discharge_ah: float = 0.0


@dataclass
class TestRunnerConfig:
    runner_tick_s: float = 2.0
    test_name: str = "unnamed"


DÖNTÉS
======

Jóváhagyom ezt az irányt.

A TestRunner osztály mehet erre az alapra.

Két ajánlott finomítás:

  1. Enumok explicit string value-val legyenek.
  2. TestStep és TestPlan lehet frozen=True, mert ezek futás közben ne módosuljanak.

Ezekkel a struktúra stabil, logbarát és jól illeszkedik a FÁZIS 5-höz.
