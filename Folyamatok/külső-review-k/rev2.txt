PRIORITÁSI JAVASLAT
===================

1. HOTFIX-1 — integrációs időalap javítása
------------------------------------------

Cél:
  Az Ah/Wh integrátor ne a névleges runner_tick_s értékkel számoljon, hanem a tényleges
  eltelt idővel.

Javítás iránya:

  loop_start = time.perf_counter()

  controller.advance(dt_s=actual_dt_s)

  elapsed = time.perf_counter() - loop_start
  sleep_s = max(0.0, runner_tick_s - elapsed)
  time.sleep(sleep_s)

Fontos:
  A pontos implementációt úgy kell kialakítani, hogy az első tick se kapjon fals dt-t.
  Én külön last_tick_time alapú megoldást választanék:

    now = time.perf_counter()
    dt_s = now - last_tick_time
    last_tick_time = now

  Így a dt tényleg a két advance közötti valós idő.

Teszt:
  Mock sleep / perf_counter.
  Elvárás:
    ha a ciklus valós ideje 2.4 s, akkor az integrátor 2.4 s-t kapjon, ne 2.0 s-t.


2. HOTFIX-2 — charge limit check minden töltési fázisban
--------------------------------------------------------

Cél:
  MAX_CHARGE_AH_REACHED és MAX_CHARGE_TIME_REACHED ne csak CC-ben fusson.

Javítás:
  Ugyanazt a limitellenőrzést hívd meg:

    _run_charge_cc()
    _run_charge_cv()
    _run_taper_hold()

Én nem másolnám három helyre hosszú távon, hanem csinálnék egy kis belső metódust:

  _check_charge_limits()

De ha gyors hotfix kell, a copy is elfogadható.

Teszt:
  Controller CV állapotba állítva.
  accumulated_charge_Ah > nominal * max_charge_Ah_factor.
  Elvárás:
    FAULT / MAX_CHARGE_AH_REACHED.


3. HOTFIX-3 — BY550 u_drop / diode_power safety bekötése
--------------------------------------------------------

Cél:
  A SafetyManager.check_series_drop() és check_diode_power() ne csak definiálva legyen,
  hanem töltés közben fusson is.

Javítás iránya:

  CHARGE_CC és CHARGE_CV alatt:

    u_drop_V = psu_readback_voltage_V - u_batt_V

    result = safety.check_series_drop(u_drop_V)
    if result.fault:
        emergency_stop(...)

    result = safety.check_diode_power(i_charge, u_drop_V)
    if result.warning:
        warning_flags/log event

Fontos:
  check_diode_power jelenlegi döntés szerint warning-only lehet, de check_series_drop
  fault legyen.

Teszt:
  Mock PSU voltage = U_batt + 1.30 V.
  Elvárás:
    SERIES_DROP_TOO_HIGH fault.


4. HOTFIX-4 — PSU current readback hiba ne legyen 0.0A
------------------------------------------------------

Cél:
  Kommunikációs hiba ne nézzen ki taper áramnak.

Jelenlegi veszély:
  PSU readback exception → 0.0 A
  0.0 A <= taper_current_A
  TAPER_HOLD elindulhat
  CHARGE_DONE hamisan létrejöhet

Javítás:
  _read_psu_current() exception esetén:

    - _dmm_valid-hoz hasonlóan külön psu_current_valid flag
    - vagy azonnali emergency_stop("PSU_COMM_LOST")
    - semmiképp ne return 0.0

Első hotfixként:
  raise vagy emergency_stop PSU_COMM_LOST.

Teszt:
  MockPSU.measure_output_current() raises.
  Elvárás:
    FAULT, nem CHARGE_DONE.


5. HOTFIX-5 — PRECHECK DMM hiba pontos fault
--------------------------------------------

Cél:
  Ha DMM nem olvasható PRECHECK-ben, akkor ne DEEPLY_DISCHARGED legyen a hiba.

Javítás:
  PRECHECK előtt:

    if not self._dmm_valid:
        emergency_stop("DMM_FEEDBACK_LOST_AT_PRECHECK")

vagy meglévő FaultCode hiányában:

    DMM_FEEDBACK_LOST

Teszt:
  INIT/PRECHECK alatt DMM exception.
  Elvárás:
    DMM fault, nem mélykisült akku.


6. HOTFIX-6 — checkpoint atomikus írás
--------------------------------------

Cél:
  checkpoint.json ne sérüljön félbeíráskor.

Javítás:

  tmp_path = checkpoint_path.with_suffix(".json.tmp")
  tmp_path.write_text(...)
  os.replace(tmp_path, checkpoint_path)

Windows alatt is jó irány.

Teszt:
  Minimum unit teszt:
    write_checkpoint után valid JSON.
  Kill-szimuláció nehezebb, de legalább az atomic write path legyen tesztelve.
